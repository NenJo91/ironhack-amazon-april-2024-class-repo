Grid

Background

CSS Grid is a two-dimensional layout system based on grids that, compared to any past web layout system, completely changes the way we design user interfaces. CSS has always been used to layout our web pages, but it has never done a good job. First, we used tables, then floats, positioning, and inline-block, but all these methods were essentially hacks and left out a lot of important functionality (vertical centering, for example). Flexbox is also a great layout tool, but its one-dimensional flow has different use cases - and they actually work great together! Grid is the first CSS module created specifically to solve the layout problems we've all been hacking around while making websites all this time.

Before diving into code, it's important to understand the terminology. Since the terms involved in this case are somewhat similar from a conceptual standpoint, it's easy to confuse them if their meanings defined by the Grid specification aren't memorized first. But don't worry, there aren't many.

Grid Container

The element on which display: grid is applied. It's the direct parent of all grid items. In this example, the container is the grid container.

Grid Item

The children (direct descendants) of the grid container. In this case, the grid elements are part of it, but sub-elements are not.

Grid Line

These are the dividing lines that make up the grid structure. They can be vertical ("column grid lines") or horizontal ("row grid lines") and reside on both sides of a row or column. In this case, the yellow line is an example of a column grid line.

Grid Cell

The space between two adjacent row grid lines and two adjacent column grid lines. It's a single "unit" of the grid. Here's the grid cell between row grid lines 1 and 2, and column grid lines 2 and 3.

Grid Track

The space between two adjacent grid lines. You can think of them as the columns or rows of the grid. Here's the grid track between the second and third row grid lines.

Grid Area

The total space surrounded by four grid lines. A grid area can be composed of any number of grid cells. Here's the grid area between row grid lines 1 and 3, and column grid lines 1 and 3.

Properties for the parent element - (grid container)

  • display

    Defines the element as a grid container and establishes a new grid formatting context for its contents.

    Properties

    • grid - generates a block-level grid
    • inline-grid - generates an inline-level grid

  • grid-template-columns & grid-template-rows

    Defines the columns and rows of the grid with a space-separated list of values. The values represent the track size, and the space between them represents the grid line.

      Values

    • Track size

      Can be a length, a percentage, or a fraction of the free space of the grid using the fr unit.
      (fr = fraction.)

    • Line name

      An arbitrary name of your choice.

    Note that a line can have more than one name. For example, here the second line will have two names: row1-end and row2-start. Let's look at the CSS example with the class container-example-more-than-one-name.

    If your definition contains parts that repeat, you can use the repeat() notation to streamline things. Let's look at the CSS example with the class container-example-repeat-notation.

    This example with the class container-example-repeat-notation is equivalent to the example with the class container-example-without-repeat-notation.

    The fr unit allows you to set the size of a track as a fraction of the free space of the grid container. For example (container-example-fractions), this will set each item to one-third of the width of the grid container. Let's look at the CSS example with the class container-example-fractions.

    Free space is calculated after any non-flexible items. In this example, the total amount of free space available for fr units doesn't include the 50px.

  • grid-template-areas

    Defines a grid template by referencing grid area names specified with the grid-area property. Repeating the name of a grid area causes the content to span those cells. A period signifies an empty cell. The syntax itself provides a visualization of the grid structure.

      Values

    • Grid area name

      Name of a grid area specified with grid-area

    • None

      No grid areas defined

    This example with the class container-example-grid-template-areas will create a grid four columns wide by three rows high. The entire top row will be composed of the header area. The middle row will consist of two main areas and a sidebar area. The last row is the entire footer area.

    Each row in your declaration must have the same number of cells.

    You can use any number of adjacent periods to declare a single empty cell. As long as the periods don't have spaces between them, they represent a single cell.

    Note that you're not naming lines with this syntax, only areas. When using this syntax, the lines at the edges of the areas are automatically named. If your grid area name is foo, the name of the starting row line and starting column line of the area will be foo-start, and the name of its ending row line and ending column line will be foo-end. This means that some lines may have multiple names, like the leftmost edge line in the previous example, which will have three names: header-start, main-start, and footer-start.

  • grid-template

    A way to abbreviate setting the grid template rows (grid-template-rows), grid template columns (grid-template-columns), and grid template areas (grid-template-area) of the grid in a single declaration.

      Values

    • None

      Sets all three properties to their initial values

    • grid-template-rows / grid-template-columns

      Sets the specified values of grid-template-columns and grid-template-rows, respectively, and sets the value of grid-template-areas to none

Properties for the child elements - (grid items)

  • grid-column-start grid-column-end grid-row-start grid-row-end

    Determines the location of a grid item by referencing specific grid lines. grid-column-start/grid-row-start is the line where the item starts, and grid-column-end/grid-row-end is the line where the item ends.

    If a grid column end/grid row end is not declared, the item will span 1 track by default.

    Items can overlap each other. You can use the z-index to control their stacking order.

      Values

    • Line

      Can be a number to refer to a numbered grid line, or a name to refer to a named grid line.

    • Span - number

      The item will span the indicated number of grid lines.

    • Span - name

      The item will extend until reaching the next line with the indicated name.

    • auto

      Indicates auto-placement, an automatic span, or a default span of one.
