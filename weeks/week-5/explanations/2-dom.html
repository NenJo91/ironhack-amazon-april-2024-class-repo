What are browser events?

An event refers to an action within the program.

Let's focus on events in the context of the browser.

Essentially, an event is an indicator that shows you certain actions have occurred so that the user can respond or take action based on the event :)

Event Handlers

As we've said before, we monitor events so that when we receive any notification that an event has occurred, the program can respond accordingly.

This action theoretically works based on functions that colloquially we call eventHandlers which also refer to eventListeners. If an event occurs and the eventHandler is invoked, we can say that the event is registered.

Let's add a button and add a click handler

Example: if we have a button with an id of "btn", the eventHandler is invoked with an alert in the browser that we pass the text "Button has been clicked". We are assigning an "onClick" property to the function which would be the eventHandler.

Assigning Event Handlers

we have several ways to assign eventHandlers to our html elements

In theory, the easiest way to add them is inline as we have seen in the previous example but it is the least recommended

Note that it is not common to find HTML event attributes like onClick, onChange, or onSubmit because html attributes are not case sensitive. Basically it is syntactically correct to use "onclick", "onClick" or "ONCLICK".

Using an eventListener and passing a parameter/argument

It is important to understand that the function we call function(e) in our JS code, is what we call a callback, we will see this later.

The (e) in that function is an object that refers to the event itself. Within that object you have access to many useful properties and functions, such as the button or mouse key that was pressed, or information about the target of the event: the DOM node that was clicked.
